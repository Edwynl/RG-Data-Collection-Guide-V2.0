File Format & Encoding Essential

CSV format (Comma-Separated Values)

File must be saved as .csv — not .xlsx, .xls, or any other format.

UTF-8 character encoding

Ensure UTF-8 encoding when exporting. Do NOT use ANSI, GB2312, or other encodings.

Period (.) as decimal separator

Use 28.5 not 28,5. Comma is the column delimiter in CSV.

No formulas or calculated cells

Export raw data values only — remove all spreadsheet formulas before saving.

File size within limits or compressed

Files >100 MB must be zipped or split by quarter (Q1–Q4). Typical: 50–500 MB for 12 months.

File naming convention

Format: SiteName_StartDate_EndDate_Type.csv — e.g., MelbourneOffice_20230101_20231231_HVAC.csv

Timestamps & Data Intervals Essential

Timestamp column present as first column

Column name: Timestamp. Must be in the first column of the CSV.

Timestamp format: YYYY-MM-DD HH:MM:SS

24-hour format required. Example: 2024-07-15 14:30:00

Consistent time interval throughout file

5-minute intervals are ideal; 15 minutes is the minimum accepted. Maximum: 30 min.

No timestamp gaps longer than 2 hours

Isolated missing rows are acceptable, but continuous gaps >2 hours must be documented.

Timestamps are sequential and non-duplicate

No duplicate timestamps. No reversed or out-of-order rows.

Data covers minimum 12 months

Must include at least one full summer peak season (Dec–Feb Southern Hemisphere / Jun–Aug Northern).

Data Completeness Essential

Overall data completeness > 80%

Missing data must be less than 20% of total records for each key column.

Missing data marked as blank or "NULL"

Do NOT use 0 (zero) for missing sensor readings — this creates false data points.

No flat-lining (constant value for long periods)

If a sensor reads the same value for more than 2 hours, check if it is a sensor fault.

No negative values in non-negative columns

Power (kW), flow (m³/h), temperatures above absolute zero. Flag anomalies in notes.

Values within physically reasonable ranges

Chiller supply temp: 5–15°C. Power: 0–5000 kW. Flow: 0–2000 m³/h. Outdoor temp: –10 to 50°C.

Essential Data Points Included (★★★) Essential

Outdoor_Temp_C — Outdoor dry-bulb temperature

Key driver for cooling load. Must not be missing during peak summer months.

Outdoor_RH_% — Outdoor relative humidity

Required for psychrometric calculations and cooling tower analysis.

Total_HVAC_Energy_kWh — Total electrical consumption

Whole-system energy consumption. Used for baseline and savings calculation.

Chiller status & power for each unit

Chiller1_Status (ON/OFF) + Chiller1_Power_kW. Repeat for each chiller unit.

Chilled water supply & return temperatures

Chiller1_CHW_Supply_Temp_C and Chiller1_CHW_Return_Temp_C for each chiller.

CHW pump status & frequency (if VSD fitted)

CHW_Pump1_Status + CHW_Pump1_Frequency_Hz. Critical for variable flow analysis.

Cooling tower fan status & frequency

CT1_Fan_Status + CT1_Fan_Frequency_Hz + CT1_Leaving_Water_Temp_C.

Column Headers & Naming Important

All column headers are in English

No Chinese, Japanese or other language headers. Translate or rename before submission.

Unit included in column name

Use suffixes: _C for °C, _kW for kW, _m3h for m³/h, _% for %. Example: Chiller1_Power_kW

Underscores instead of spaces in header names

Use Outdoor_Temp_C not Outdoor Temp C. Spaces can break automated parsing.

Numbered suffixes for identical equipment

Use Chiller1, Chiller2, CHW_Pump1, CHW_Pump2 etc. — not just "Chiller" for multiple units.

Status columns use consistent 1/0 or ON/OFF

Do not mix notation. 1=ON, 0=OFF is preferred. "True/False" is not accepted.

Pre-Submission Documents Important

Data Collection Form completed (all 4 sheets)

Sheet 1: Project Info · Sheet 2: Equipment · Sheet 3: BMS Info · Sheet 4: Data Requirements

Historical CSV file(s) prepared

Covering minimum 12 months. If >100 MB, split by quarter or compress as ZIP.

Known issues or anomalies documented

Note any system shutdowns, sensor faults, or maintenance periods inside the submission email.

Point mapping document (if internal naming used)

If your BMS uses internal tag names, provide a mapping table: BMS tag → standard column name.
